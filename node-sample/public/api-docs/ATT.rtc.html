Namespace: rtc

rtc

This namespace provides the methods necessary to configure the JS client library.

  • You must configure the JS client library using configure method before you can use the ATT.rtc.createAccessToken or ATT.rtc.createE911Id method.
  • You may use createAccessToken method to create an access token for any user type
  • You may use createE911Id method to create an E911 Id for any mobile number or virtual number user
  • You can use the associateAccessToken method to associate an access token with a virtual number or account id user.

<static> configure(options)

Setup the endpoints for token and e911 operations

Use this method to configure the endpoints for token and e911 operations that are used by ATT.rtc.createAccessToken and ATT.rtc.createE911Id methods. Optionally you may also configure AT&T's enhanced WebRTC API endpoint and uri.

Parameters:
Name Type Description
options Object

SDK configuration options.

Properties
Name Type Argument Description
token_endpoint String

A RESTful [POST] web server endpoint URL for creating access token.
e911_endpoint String

A RESTful [POST] web server endpoint URL for creating e911 id.
api_endpoint String <optional>

The Enhanced WebRTC API URL. The default is already configured.
ewebrtc_uri String <optional>

The Enhanced WebRTC API URI. The default is already configured.
Examples
// Configure Token and E911 endpoints
ATT.rtc.configure({
  token_endpoint: '/tokens',
  e911_endpoint: '/e911ids'
});
// Configure Token, E911 and API endpoints
ATT.rtc.configure({
  token_endpoint: '/tokens',
  e911_endpoint: '/e911ids',
  api_endpoint: 'https://api.att.com'
});
// Configure Token, E911 and API endpoints
ATT.rtc.configure({
  token_endpoint: '/tokens',
  e911_endpoint: '/e911ids',
  api_endpoint: 'https://api.att.com',
  ewebrtc_uri: '/RC/v1'
});

<static> createAccessToken(options)

Create an Access Token using AT&T's OAuth for mobile number, virtual number and account id users.

This methods accepts a app_scope and creates the access token for that particular app_scope. Accepted values for app_scope are: MOBILE_NUMBER, VIRTUAL_NUMBER, ACCOUNT_ID and E911.

For mobile number user, the method requires an additional param auth_code for the authorization code obtained during the consent flow.

Parameters:
Name Type Description
options Object
Properties
Name Type Argument Description
app_scope String

Application scope for getting access token
auth_code String <optional>

Authorization Code from user consent for mobile number user.
success ATT.rtc.accessTokenSuccess

The callback that handles the succesful response after creating an AT&T OAuth Access Token.
error ATT.rtc.createTokenError

The callback that handles the generated error.
Examples
// Create access token using DHS
var success = function () {...};
var error = function () {...};
ATT.rtc.createAccessToken({
  app_scope: 'E911',
  success: success,
  error: error
});
// Create access token using DHS
var success = function () {...};
var error = function () {...};
ATT.rtc.createAccessToken({
  app_scope: 'MOBILE_NUMBER',
  auth_code: 'auth_code_from_consent_flow'
  success: success,
  error: error
});

<static> createE911Id(options)

Creates a new E911 Id using an `e911` scope access token and an address

E911 Id is required for mobile number and virtual number user for creating enhanced webrtc session. Use this function to create an E911 Id that can be used for creating the enhanced webrtc session for mobile number and virtual number users. This method requires an access token obtained using E911 auth_scope and a physical address with these string fields:

  • first_name

  • last_name

  • house_number

  • street

  • unit (optional)

  • city

  • state

  • zip

  • is_confirmed (true or false)

Example

var error = function () {...};

  token: 'e911_id_scoped_access_token',
  address: {
   first_name: 'John',
   last_name: 'Doe',

  },
